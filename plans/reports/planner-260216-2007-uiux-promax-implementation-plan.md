## Key Architecture Decisions

### 1. Remove Dark Mode Entirely
**Rationale:** Simplifies codebase (240+ dark: classes), light-only with section rhythm achieves visual variety
**Impact:** Breaking change, but acceptable since content regenerated by rewriter
**Benefit:** -342 occurrences, cleaner CSS, faster development

### 2. Hybrid Color Palette
**Primary:** Teal-green (hue 178) — nature, brand continuity
**Secondary:** Blue-grey (hue 240) — trust, professionalism for B2B
**Rationale:** Bridges brand identity (teal) with industry credibility (blue-grey)

### 3. Component Catalog Contract
**Pattern:** Template implements → Catalog documents → Rewriter follows
**Format:** Markdown with structured sections (Purpose, Props, Variants, Usage, Rules)
**Audience:** Dual (human developers + AI agents)
**Benefit:** Single source of truth prevents template-rewriter drift

### 4. Tier-Based Component Upgrade
**Tier 1:** 10 high-impact (full redesign, user-facing)
**Tier 2:** 10 medium-impact (significant polish, frequent use)
**Tier 3:** 27 low-impact (consistency pass)
**Rationale:** Prioritizes visible components first, allows parallel work

### 5. CSS-Only Premium Effects
**Gradient text:** `bg-clip-text` + `text-transparent` (no JS)
**Hover lift:** `hover:-translate-y-1` + `transition-transform`
**Shimmer:** CSS `@keyframes` animation
**Rationale:** Zero JS bundles, GPU-accelerated, performant

## Risk Assessment

- Regex batch replace misses complex ternary patterns → Manual review high-density files first
- PricingTable script removal breaks pricing display → Test pricing pages specifically

### Phase 2 Risks
- Secondary color change breaks existing design intent → Review all secondary usage
- Glass utilities become invisible on white bg → Test Header backdrop blur thoroughly

- Gradient text browser support → Test Safari, Firefox, Chrome
- BentoGrid auto-rows height → May need adjustment per design

- Breaking changes affect existing content → Acceptable, content regenerated
- Hover effects degrade on touch devices → Test on real mobile device

### Phase 5 Risks
- Prose max-width breaks full-width layouts → Apply prose only to content areas

### Phase 6 Risks
- Catalog documentation drift from implementation → Update together workflow
